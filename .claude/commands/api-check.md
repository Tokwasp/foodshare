`CLAUDE.md`의 API 명세와 `.claude/docs/$1/` 하위 문서가 일치하는지 검사한다.
`CLAUDE.md`는 도메인별로 구분되어 있지 않으므로, 도메인 범위는 **docs 폴더에서
다루는 엔드포인트를 기준으로** 식별한다.

## 인자 검증
- `$1`(도메인)이 비어 있으면 검사를 진행하지 말 것.
  `.claude/docs/` 하위 폴더 목록을 보여주고
  "어떤 도메인을 검사할까요? 예: /api-check 결제" 라고 물은 뒤 중단한다.
- `.claude/docs/$1/` 폴더가 없으면 그 사실을 알리고 중단한다.

## 기준(정답) 방향 결정
- `$2`가 `spec`이면 → **CLAUDE.md가 기준**, docs를 맞춘다.
- `$2`가 `docs`이면 → **docs가 기준**, CLAUDE.md를 맞춘다.
- `$2`가 비어 있으면 검사를 시작하기 전에 사용자에게 물어본다:
  "어느 쪽을 정답(기준)으로 볼까요?
    1) CLAUDE.md 명세 기준 → docs 수정안 제시
    2) docs 문서 기준 → CLAUDE.md 명세 수정안 제시"
       답을 받은 뒤 진행한다.

## 절차
1. `.claude/docs/$1/` 하위 모든 `*.md`를 읽고, 거기서 다루는
   엔드포인트 목록(경로 + HTTP 메서드)을 먼저 수집한다.
2. 1에서 모은 경로/메서드를 `CLAUDE.md` 전체에서 검색해 대응되는
   명세 항목을 찾는다. (CLAUDE.md는 도메인 헤딩이 없으므로 경로 매칭으로 범위를 잡는다)
3. 매칭된 항목끼리 아래를 항목별로 1:1 대조한다.
    - 엔드포인트 경로, HTTP 메서드
    - 요청 파라미터 / 바디 필드 (이름, 타입, 필수 여부, 기본값)
    - 응답 필드 (이름, 타입, 구조)
    - 상태 코드 / 에러 응답
4. 불일치를 분류한다. (방향에 관계없이 "기준 → 대상" 관점으로 표기)
    - **누락**: 기준엔 있는데 대상에 없음
    - **불필요**: 대상엔 있는데 기준엔 없음
    - **불일치**: 양쪽에 있으나 값이 다름 (경로·메서드·타입·필수여부·필드명 등)
5. 각 불일치마다 **기준에 맞춘 수정안**을 제시한다.
    - 기준이 CLAUDE.md면 → docs 문서를 어떻게 고칠지
    - 기준이 docs면 → CLAUDE.md 명세를 어떻게 고칠지

## 중요 제약
- 파일을 **직접 수정하지 말 것.** 차이점과 수정안만 보고한다.
- 기준으로 정한 쪽은 "정답"으로 간주하고 수정 대상에서 제외한다.

## 출력 형식
맨 위에 이번 검사 조건을 한 줄로 명시한다.
예) `검사: 결제 도메인 | 기준: docs → 대상: CLAUDE.md`

그다음 항목별로 정리한다.
| 구분 | 엔드포인트 | 항목 | 기준 | 대상(현재) | 수정안 |
|------|-----------|------|------|-----------|--------|
| 불일치 | POST /payments | `amount` 타입 | integer(필수) | string | integer·필수로 변경 |

마지막에 전체 요약:
- 누락 N건 / 불필요 N건 / 불일치 N건
- 0건이면 "✅ $1 도메인 일치 (기준: {방향})" 만 출력
